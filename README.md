# 活动签到与桌位管理系统

## 功能概览

本系统用于**季度发表会**等大型活动的报名名单管理、**自动分桌**和**现场签到**。

### 核心功能

1. **自动分桌排序**
   - 上传报名名单（Excel/CSV）
   - 按以下优先级自动分桌：
     1. 推荐人相同 → 同一桌
     2. 班级相同 → 同一桌
     3. 分中心相同 → 同一桌
     4. 同一公司 → 同一桌（特殊情况）
     5. 特殊要求 → 同一桌
   - 自动分配**正座桌号**和**空巴（晚宴）桌号**

2. **扫码签到**
   - 签到页面输入姓名 + 手机号
   - 返回：**姓名**、**桌号**、**空巴桌号**
   - 实时统计签到率

3. **数据管理**
   - 多批次活动管理
   - 分桌结果导出为 CSV
   - 签到统计看板

## 快速开始

### 本地运行

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 启动应用
streamlit run app.py

# 3. 浏览器访问
# http://localhost:8501
```

### 部署到 Streamlit Cloud

1. 将整个 `signin_system` 文件夹推送到 GitHub
2. 登录 [share.streamlit.io](https://share.streamlit.io)
3. 选择仓库和分支，入口文件设为 `app.py`
4. 部署完成后即可获得公开 URL 用于生成签到二维码

## 使用流程

### 第一步：准备报名名单

准备一个 Excel 文件，包含以下列（列名可灵活匹配）：

| 必填列 | 选填列 |
|--------|--------|
| 姓名   | 推荐人 |
| 手机号 | 班级   |
|        | 分中心 |
|        | 公司   |
|        | 特殊要求 |

参考 `sample_data.xlsx`。

### 第二步：管理员上传并分桌

1. 打开管理页面 → 上传名单
2. 设置每桌人数（正座默认10人，空巴默认8人）
3. 点击「开始自动分桌」
4. 查看分桌结果 → 导出 CSV

### 第三步：现场签到

1. 参会者使用手机扫描二维码
2. 输入姓名 + 手机号
3. 系统返回：桌号、空巴桌号

## 技术栈

- **前端/UI**: Streamlit
- **数据库**: SQLite（本地存储）
- **核心算法**: Python 贪心分组算法
- **依赖**: pandas, openpyxl, qrcode

## 注意事项

- 手机号用于签到验证，请确保报名名单中的手机号准确
- 分桌结果可多次导出，每批次数据独立存储
- 空巴桌号按分中心 + 优化合并策略分配
